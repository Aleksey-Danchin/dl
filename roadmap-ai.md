# AI Learning Roadmap

Дорожная карта к освоению работы с ИИ — от основ до production.

---

## Блок 00 — Фундамент

> Без этого блока всё остальное — карточный домик.

| Инструмент | Зачем |
|---|---|
| Python | Язык. Нужен хорошо — классы, генераторы, типизация |
| Git | Версионирование кода. Без него нельзя работать в команде |
| Линейная алгебра | Матрицы, векторы, операции. Основа нейросетей |
| Статистика | Распределения, вероятность, проверка гипотез |

---

## Блок 01 — Базовый Data Science

> Классический ML, работа с данными, визуализация.

| Библиотека | Зачем |
|---|---|
| `numpy` ★ | Массивы, матричные операции. Всё остальное строится на нём |
| `pandas` ★ | Табличные данные, очистка, агрегация. Рабочая лошадь DS |
| `matplotlib` | Базовая визуализация. Многословно, но гибко |
| `scikit-learn` ★ | Классический ML: деревья, SVM, кластеризация, пайплайны |
| `scipy` | Статистика, оптимизация, сигналы. Логично рядом с numpy |

---

## Блок 02 — Среда разработки

> Jupyter с первого дня — это среда, в которой вы живёте.

| Библиотека | Зачем |
|---|---|
| `jupyterlab` | Интерактивная среда. Notebooks — стандарт для экспериментов в DS/ML |

---

## Блок 03 — Deep Learning

> Центр всей карты. PyTorch — де-факто стандарт.

| Библиотека | Зачем |
|---|---|
| `torch` (PyTorch) ★ | Главный DL-фреймворк. Динамические графы, гибкость, огромное комьюнити |
| `tensorboard` | Мониторинг обучения: loss, метрики, гистограммы весов. Встроен в torch |
| `pytorch-lightning` | Структурирует torch-код. Убирает boilerplate, учит паттернам |

---

## Блок 04 — Computer Vision + NLP / LLM

> Два главных прикладных направления.

| Библиотека | Зачем |
|---|---|
| `Pillow` | Базовая работа с изображениями. Часто нужна раньше OpenCV |
| `opencv-python` | Классический CV: чтение изображений, фильтры, детекция контуров |
| `albumentations` | Аугментации для CV. Быстрая, гибкая, интегрируется с torch |
| `transformers` ★ | HuggingFace. BERT, GPT, Whisper, CLIP — всё через единый API |
| `datasets` | HuggingFace datasets. Загрузка, стриминг, препроцессинг. Пара к transformers |

---

## Блок 05 — Boosting + подбор гиперпараметров

> XGBoost и LightGBM часто бьют нейросети на табличных данных.

| Библиотека | Зачем |
|---|---|
| `xgboost` | Градиентный бустинг. Стандарт для соревнований на табличных данных |
| `lightgbm` | Быстрее XGBoost на больших данных |
| `optuna` | Автоматический подбор гиперпараметров. Байесовская оптимизация |

---

## Блок 06 — Прототипирование и деплой моделей

> Быстрое демо и правильный путь к production API.

| Библиотека | Зачем |
|---|---|
| `gradio` | Веб-интерфейс для модели за 5 строк. Отлично для демо |
| `streamlit` | Альтернатива Gradio. Чуть больше свободы для дашбордов |
| `fastapi` ★ | Async REST API. Production-ready, автодока, быстрый |
| `pydantic` | Валидация данных. Неразлучна с FastAPI |
| `onnx` | Open Neural Network Exchange. Экспорт и запуск модели без фреймворка |
| `seaborn` | Красивые статплоты в 1 строку. Поверх matplotlib |
| `plotly` | Интерактивные графики. Дашборды, 3D, maps |

---

## Блок 07 — MLOps

> Production ML — это не только модель.

| Инструмент | Зачем |
|---|---|
| `mlflow` ★ | Трекинг экспериментов, реестр моделей, воспроизводимость |
| `dvc` | Git для данных и моделей. Версионирование больших файлов |
| `Docker` ★ | Контейнеры. Без него MLOps — просто слово |

---

## Блок 08 — Продвинутое / по выбору

> Нишевые инструменты — только при чётком запросе.

| Библиотека | Зачем |
|---|---|
| `jax` | NumPy + autograd + JIT + XLA. Для кастомных исследований и скорости |
| `flax` | Нейросети поверх JAX. Используется в Google Research, DeepMind |
| `tensorflow` | Второй крупный фреймворк. Полезен при TFX/TPU/legacy коде |
| `accelerate` | HuggingFace Accelerate. Multi-GPU/TPU без переписывания кода |
